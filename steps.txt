1. git init - инициализация гита в папке для загрузки данных на сервер GitHub (далее гит)
2. readme.md - создаем файл с описание нашего проекта/библиотеки
3. git add . - добавляем файлы с изменениями в загрузку на сервер гита
4. git commit -m 'Project created' - инициируем первый коммит файлов на сервер гита
5. git remote add origin https://github.com/gosha-phantom/locallibrary.git - привязываем папку с сервера гита
6. git push -u origin master - заливаем файлы на сервер гита
7. python -m venv venv - инициализируем виртуальное окружение venv
8. .gitignore - создаем файл с именами-исключениями из загрузки в гит
9. readme.md - создаем файл с описание нашего проекта/библиотеки
10. инициализируем виртуальное окружение (нет команды, как правило я создаю пустой файл main.py и активирую командную строку)
11. pip install django - устанавливаем фреймворк django
12. django-admin startproject locallibrary . - создаем новый проект django в корневой папке
13. python manage.py statrapp catalog - создаем приложение в проекте из корневой папки проекта
14. python manage.py makemigrations - при изменении структуры базы данных необходимо инициировать изменения из корневой папки проекта
15. python manage.py migrate - после инициации изменений необходимо внести изменения в бд из корневой папки проекта
16. python manage.py createsuperuser - создаем супрпользователя для работы с админкой (login=admin1, password=root1)
